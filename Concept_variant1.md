# КОНЦЕПЦИЯ СИСТЕМЫ ПОМОЩИ В ПРОВЕДЕНИИ ЛЕКЦИЙ ПРЕПОДАВАТЕЛЕЙ ВУЗОВ

## 1 Цель
### 1.1 Клиентские пути
##### Преподавателя
###### Вариант 1 
1) Преподаватель согласно расписанию готовится к проведению лекции студентам по своему курсу и определённой теме
2) Преподаватель готовит план лекции накануне или обновляет существующий вариант плана
3) Преподаватель приходит на лекцию и рассказывает по плану материал
4) По ходу лекции преподаватель вставляет свои метафоры/примеры из жизни по теме лекции
5) Преподаватель отвечает на вопросы по ходу лекции
6) По завершению лекции преподаватель отвечает на оставшиеся вопросы студентов
7) Преподаватель отправляет материал лекции студентам для домашнего ознакомления/перечитывания
###### Вариант 2
1) Преподаватель согласно расписанию готовится к проведению лекции студентам по своему курсу и определённой теме
2) Преподаватель приходит на лекцию и рассказывает материал без чёткого плана, импровизирует по ходу лекции
3) По ходу лекции преподаватель вставляет свои метафоры/примеры из жизни по теме лекции
4) Преподаватель отвечает на вопросы по ходу лекции
5) По завершению лекции преподаватель отвечает на оставшиеся вопросы студентов

##### Студента
1) Студент приходит на лекцию
2) Студент слушает лекцию преподавателя
3) По ходу лекции студент записывает ключевые моменты лекции, задаёт вопросы преподавателю по непонятным моментам в лекции
4) По оканчанию лекции студент задаёт оставшиеся вопросы преподавателю
5) Студент изучает полученные от преподавателя материала, если таковые были
### 1.2 Описание болей
##### Преподавателя
1) Большие ментальные затраты на ведение лекций без плана
2) На подготовку плана лекции необходимо время
3) Поиск метафор, подходящих примеров из жизнии по теме лекции
4) Невозможность тратить много времени на объяснение непонятных моментов в материале лекции студентам в рамках самой лекции
5) Сложно рассказать весь материал без плана, не забыв отдельные моменты

##### Студента
1) Проблема приведения и понимания метафор преподавателя
2) Если что-то не будет рассказано по материалу лекции, то студент об этом не узнает
3) Мотивация слушать лекцию
4) Невозможность задать вопрос или выявить ключевые моменты для конспекта, так как непонятен язык лекции

### 1.3 Выявление проблем
##### Преподавателя
1) Преподавателю необходимо делать план лекции для максимально объёма покрытия материала лекции
2) Преподавателю необходимо выявлять удачные метафоры и "сохранять" их на будущие схожие лекции
3) Преподавателю необходимо максимально сократить время на подготовку плана лекции, без ущерба в качестве проведения лекции по нему

##### Студента
1) Студенту необходимо помечать "проблемные" места в лекции, чтобы в будущем в них разобраться
2) Студенту необходимо упростить создание конспекта лекции
3) Студенту необходимо иметь возможность повторно прослушать лекцию для лучшего понимания материала

### 1.4 Формирование задач и целей
##### Задача - разработать веб-приложение для помощи в проведении лекций для преподавателей университетов в течении 6 месяцев, которое сможет обеспечить запись лекций, пометку отдельных участков записи и автоматическое составление конспекта/плана по записанному материалу
##### Цели:
1) Разработать модуль приложения отвечающий за аудио запись лекции
2) Разработать модуль анализа и обработки записи лекции на предмет ключевых моментов
3) Разработать модуль пометки участков записи лекции, к которым можно будет быстро вернуться при повторном прослушивании

#### Уровень 1: Контекстная диаграмма (Система и её окружение)
##### Основные компоненты:
1) Преподаватель - основной пользователь системы
2) Студенты - получатели материалов и участники интерактива
3) Облачное хранилище - внешние сервисы для хранения материалов
4) физическое хранилище - физическая машина для хранения записей лекций и их обработок

##### Ключевые взаимодействия:
Преподаватель проводит лекции и делает её аудиозапись
Студенты получают материалы и помечают проблемные, интересные места
Система делает обработку записи на предмет ключевых моментов и помеченных мест
Преподаватель и студенты получают обработанный текстовый вариант лекции
