### Слоистая архитектура приложения

Приложение построено по слоистой архитектуре с чётким разделением ответственности между слоями.

1. **Слой домена (`app/domain`)**
   - Файл `models.py` содержит доменные сущности:
     - `Lecture` — лекция с полями `id`, `title`, а также необязательными ссылками на текст, аудиозапись и конспект.
     - `NoteMark`, `ProblemMark` — отметки по заметкам и проблемам, привязанные ко времени.
   - Файл `repositories.py` определяет абстрактный интерфейс `LectureRepository`, не зависящий от конкретной СУБД или NocoDB. Через него реализуются методы:
     - `get_lecture(lecture_id: int) -> Lecture`
     - `list_lectures_by_date(lecture_date: date) -> List[Lecture]`

2. **Инфраструктурный слой (`app/infrastructure`)**
   - `db_interface.py` задаёт абстрактный интерфейс `DatabaseInterface` для низкоуровневой работы с БД (методы `execute`, `fetch_one`, `fetch_all`, `transaction`).
   - `sqlite_database.py` реализует `DatabaseInterface` поверх `sqlite3`, инкапсулируя подключение к файлу SQLite и работу с транзакциями.
   - `lecture_repository_sqlite.py` реализует интерфейс `LectureRepository` для таблицы лекций в SQLite:
     - Преобразует строки из таблицы `nc_9ivt__Лекции` в объекты `Lecture`.
     - Реализует выбор лекции по ID и выбор списка лекций по дате проведения.
   - Отдельные компоненты инфраструктуры (NocoDB‑клиент и репозитории для работы через NocoDB) используются веб‑диаграммой и не зависят от Tk‑приложения напрямую.

3. **Сервисный слой (`app/services`)**
   - `lecture_service.py` реализует прикладной сервис `LectureService`, который:
     - Принимает реализацию `LectureRepository` (SQLite или NocoDB).
     - Предоставляет методы `get_lecture` и `list_lectures_by_date` для использования в пользовательском интерфейсе.
   - Сервисный слой инкапсулирует бизнес‑логику и правила работы с лекциями, оставаясь независимым от конкретной БД и UI.

4. **Пользовательский интерфейс (`app/ui_tk.py` и веб‑диаграмма)**
   - `ui_tk.py` реализует настольное Tkinter‑приложение `LectureApp`:
     - Инициализирует `SqliteDatabase`, `LectureRepositorySqlite` и `LectureService`.
     - Создаёт главное окно с:
       - Поиском лекции по ID.
       - Поиском лекций по дате в формате `ГГГГ-ММ-ДД`.
       - Списком найденных лекций и областью подробностей.
       - Кнопкой выхода и обновлением списка лекций по дате.
   - Файл `Information/DiagrammOfMonth.py` реализует отдельное веб‑приложение на Dash:
     - Подключается к NocoDB.
     - Собирает данные о лекциях и строит диаграмму количества занятий по дням месяца.

5. **Связи между слоями**
   - UI‑слой (`LectureApp` в `ui_tk.py`) работает только с `LectureService` и не обращается к БД напрямую.
   - `LectureService` зависит от абстракции `LectureRepository` и не знает о конкретной реализации (SQLite или NocoDB).
   - Конкретная реализация `LectureRepositorySqlite` использует `DatabaseInterface`, реализованный `SqliteDatabase`, для выполнения SQL‑запросов к таблице лекций.
   - Такое разделение позволяет при необходимости заменить источник данных (например, с SQLite на NocoDB) с минимальными изменениями в сервисном слое и без изменений в доменной модели.
