### Схема таблицы лекций в SQLite

Для настольного приложения на Tkinter используется база данных SQLite с таблицей лекций `nc_9ivt__Лекции`. По коду репозитория она как минимум содержит следующие поля:

- **`id`** — целочисленный первичный ключ (идентификатор лекции).
- **`Тема_лекции`** — текст (название/тема лекции).
- **`Дата_проведения`** — дата проведения лекции (используется для поиска по дате).

Пример DDL‑описания таблицы (может отличаться от фактической структуры, но отражает ключевые поля, которые использует приложение):

```sql
CREATE TABLE IF NOT EXISTS "nc_9ivt__Лекции" (
    "id"              INTEGER PRIMARY KEY AUTOINCREMENT,
    "Тема_лекции"     TEXT NOT NULL,
    "Дата_проведения" TEXT NOT NULL -- дата в формате 'YYYY-MM-DD'
);
```

### Соответствие полей доменной модели

Домашняя модель `Lecture` (`app/domain/models.py`) включает следующие поля:

- `id: int` — соответствует полю `id` в таблице.
- `title: str` — заполняется из поля `"Тема_лекции"`.
- `audio_record_url`, `text`, `conspect_url` — в текущей реализации SQLite‑репозитория не заполняются (хранятся как `None`), но могут быть добавлены в БД при дальнейшем развитии приложения.

### Использование в сценариях MVP

Для реализованных сценариев MVP настольного приложения достаточно полей:

- `id` — для прямого поиска лекции по идентификатору.
- `Тема_лекции` — для отображения понятного названия лекции в списке и деталях.
- `Дата_проведения` — для поиска и отображения всех лекций, проведённых в указанную дату.
