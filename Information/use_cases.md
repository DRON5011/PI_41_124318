### Назначение приложения

Приложение предназначено для **просмотра и анализа расписания и содержания лекций**, хранящихся в базе данных (SQLite/NocoDB). Пользователь может находить лекции по идентификатору и дате проведения, просматривать их основные сведения и использовать отдельное веб‑приложение для визуализации статистики по дням месяца.

### Пользовательские сценарии (MVP)

1. **Просмотр лекции по ID**
   - Пользователь запускает настольное приложение.
   - Вводит идентификатор лекции в поле «ID лекции».
   - Нажимает кнопку «Показать».
   - В правой части окна отображаются подробности лекции (ID, тема и сопутствующая информация).

2. **Поиск и просмотр лекций по дате**
   - Пользователь вводит дату в формате `ГГГГ-ММ-ДД` в поле «Дата (ГГГГ-ММ-ДД)».
   - Нажимает кнопку «Найти лекции по дате».
   - В списке слева отображаются все лекции, найденные на указанную дату.
   - Пользователь выбирает лекцию из списка и видит её подробности в правой части окна.

3. **Обновление списка лекций на выбранную дату**
   - После изменения даты или обновления данных в БД пользователь нажимает кнопку «Обновить по дате».
   - Приложение повторно загружает лекции на выбранную дату и обновляет список и область подробностей.

4. **Просмотр статистики занятий по дням месяца (веб‑диаграмма)**
   - Пользователь запускает отдельное веб‑приложение на основе Dash.
   - Приложение загружает данные о занятиях из NocoDB и агрегирует их по дням месяца.
   - На экране отображается столбчатая диаграмма, показывающая количество занятий по дням месяца.
   - При ошибке загрузки данных пользователь видит текстовое сообщение об ошибке и пустую диаграмму.
