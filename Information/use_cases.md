### Назначение приложения

Приложение предназначено для **Сбора данных по лекциям, с возможностью подгружать к ним аудиофайлы(В БД гиперссылки на эти файлы, также должна быть система пользователей(студенты/преподаватели) и к лекциям могут быть добавлены места интереса(проблемные места/метафоры), что хранятся в отдельных одноимённых таблицах)**, хранящихся в базе данных (SQLite/NocoDB). Пользователь может находить лекции по идентификатору и дате проведения, просматривать их основные сведения и использовать отдельное веб‑приложение для визуализации статистики по дням месяца.

### Пользовательские сценарии (MVP)

1. **Регистрация пользователя**
   - Пользователь запускает настольное приложение.
   - Пользователь нажимает кнопку "Добавить нового пользователя"
   - Заполняет окошки ФИО, почта, пароль и из выпадающего списка выбирает студент/преподаватель.
   - нажимает кнопку добавить и данные попадают в БД.

2. **Авторизация пользователя**
   - Пользователь запускает настольное приложение.
   - Пользователь нажимает кнопку "Войти"
   - В появивишихся окошках вводит почту и пароль, а также выбирает из списка тип студент/преподаватель.
   - Пользователь нажимает войти и приложение сверяется с БД (студентов или преподавателей), если пользователь есть в БД заходит в приложение под ним.
   - Пользователь попадает на начальный экран.

3. **Просмотр лекций**
   - Авторизованный пользователь нажимает кнопку посмотреть лекции и в приложениее открывается окно со списком всех лекций из БД.
   - Пользователь может указать фильтр по одному из столбцов, тогда система делает повторный запрос к БД и отбирает только подходящие под фильтр записи.
   - Пользователь может нажать кнопку "Вернуться" и тогда приложение открывает начальный экран

4. **Добавление лекции(может делать только преподаватель)**
   - Авторизованный как преподаватель пользователь имеет дополнительную кнопку "добавить лекцию".
   - При нажатии "Добавить лекцию" приложение открывает окно в котором есть окошки для заполнения: Тема лекции, Название курса, Дата проведения, Время начала, Время оканчания. Название курса и Дата проведения обязательные окна.
   - После заполнения данных разблокируется кнопка "Добавить"
   - При нажатии кнопки "Добавить" данные передаются в БД и пользователя возращает к начальному экрану.
